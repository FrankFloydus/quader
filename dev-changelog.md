# Dev Changelog

This file records durable development changes to the repository. It is the
engineering log for agents and maintainers, not user-facing release notes.

Use newest-first entries under dated sections. Record meaningful source, docs,
workflow, build, architecture, tooling, task-board, verification, and maintenance
changes. Temporary smoke-test edits that are fully reverted before handoff do not
need entries.
